# Pebbles Sketch

## What Is Being Drawn?
This sketch builds a geometric composition of rectangular blocks that are progressively drawn and filled by hand-like strokes.  
The structure appears first as dark construction lines, then each block is colored with dense angled hatching so the image feels painted rather than flood-filled.

Visually, the piece balances strict grid logic with softer edges:
- Rectangles are generated by recursive splitting, so the layout feels intentional and architectural.
- Every block gets rounded corners, which removes the rigid digital look.
- Fills are clipped to the rounded interior, so color respects curved boundaries instead of bleeding into corner cutouts.
- The drawing order goes from top-left toward bottom-right, preserving a human "working across the page" rhythm.

## What Parameters Are Used?
All exact active values live in constants at the top of `pebbles/sketch.js`. Tune there; keep this document behavior-focused.

- Layout size and proportions:
  `AREA_MIN_RATIO`, `AREA_MAX_RATIO`, `RECT_RATIO_MIN`, `RECT_RATIO_MAX` control how large sections can be and how stretched they may become.
- Framing and spacing:
  `COMPOSITION_MARGIN_RATIO`, `COMPOSITION_MARGIN_MIN`, `COMPOSITION_MARGIN_MAX` control how far the composition sits from the canvas edges.
- Split search and fallback behavior:
  `MAX_LAYOUT_ATTEMPTS`, `MAX_LAYOUT_GUARD_STEPS`, `SPLIT_ORIENTATION_ATTEMPTS`, `FALLBACK_GRID_ROWS`, `FALLBACK_GRID_COLUMNS` control retry budgets and fallback grid shape when a valid split layout is hard to find.
- Fill direction and density:
  `FILL_ANGLE_MIN_DEGREES`, `FILL_ANGLE_MAX_DEGREES`, `FILL_SPACING_MIN_PX`, `FILL_SPACING_MAX_PX`, `FILL_SPACING_MIN_SCALE`, `FILL_SPACING_MAX_SCALE` control hatching direction and line density.
- Stroke reveal pacing:
  `PIXEL_STEP_DISTANCE`, `HUMAN_PIXEL_STEPS_PER_FRAME` and app speed profile hooks control progressive drawing at `1x`, `2x`, and `4x`.
- Corner geometry:
  `CORNER_RADIUS_MIN_SCALE`, `CORNER_RADIUS_MAX_SCALE`, `ARC_SEGMENT_MIN_LENGTH` control rounded-corner shape and arc smoothness.
- Fill-boundary clipping:
  `FILL_CLIP_SAMPLE_STEP` controls sampling density when clipping line segments to rounded interiors.
- Color behavior:
  `RED_PASTEL_SHADES`, `YELLOW_PASTEL_SHADES`, `BLUE_PASTEL_SHADES`, `WHITE_RECTANGLE_PROBABILITY`, `BLACK_FILL_PROBABILITY`, `WHITE_FILL_COLOR`, `BLACK_FILL_COLOR` control palette selection and white/black frequency.
- Line and background appearance:
  `GRID_COLOR`, `BG_COLOR`, `FILL_STROKE_WEIGHT_MULTIPLIER` control structural line look and fill texture weight.

## Drawing Strategy
1. Initialize a margin-bounded composition region and derive area constraints from current canvas size.
2. Recursively split oversized regions until all blocks satisfy area and aspect-ratio rules.
3. If splitting fails repeatedly, fall back to a deterministic grid so the sketch always completes.
4. For each final block, build a fill plan: rounded corner radii, inset fill radii, fill style, hatch angle, and clipped line segments.
5. Build outline segments (including corner arcs) for every block.
6. Animate drawing in two phases:
   first outlines, then interior fill strokes.
7. During fill drawing, clip each stroke both geometrically (segment generation) and at render time (canvas clip path) to rounded interiors.
8. Respect global speed mode from `window.HUAHUA_APP` and stop when all planned segments are complete.

## Algorithms
### 1) Recursive Partitioning With Retry + Fallback
Generate valid sections by repeatedly splitting the largest oversized section. If retries are exhausted, use a fallback grid.

```js
for (let attempt = 0; attempt < MAX_LAYOUT_ATTEMPTS; attempt += 1) {
  const sections = [root];
  let failed = false;
  let guard = 0;

  while (true) {
    guard += 1;
    if (guard > MAX_LAYOUT_GUARD_STEPS) {
      failed = true;
      break;
    }
    // find largest oversized section and split it...
  }

  if (!failed && sections.every((s) => isSectionFinalValid(s, minArea, maxArea))) return sections;
}
return createFallbackGrid(root);
```

### 2) Per-Section Fill Plan (Rounded + Inset-Aware)
Each section stores both outline corner radii and inset fill corner radii, then precomputes fill lines.

```js
const cornerRadii = pickRandomCornerRadii(section.w, section.h);
const fillCornerRadii = getInsetCornerRadii(cornerRadii, inset, fillW, fillH);
const lines = fillStyle.skipFill
  ? []
  : createSolidFillStrokes(section, lineSpacing, inset, fillAngleDegrees, fillCornerRadii);
```

### 3) Rounded Outlines From Line + Arc Segments
Outline paths are assembled from straight edges plus quarter-arc approximations for each corner.

```js
pushSegment(segments, x + tl, y, x + w - tr, y);
addArcSegments(segments, x + w - tr, y + tr, tr, -HALF_PI, 0);
pushSegment(segments, x + w, y + tr, x + w, y + h - br);
addArcSegments(segments, x + w - br, y + h - br, br, 0, HALF_PI);
```

### 4) Hatch Fill Generation With Rounded Segment Clipping
Candidate hatch lines are clipped to the inset rectangle, then clipped again to the rounded boundary.

```js
const clipped = clipSegmentToRect(rawX1, rawY1, rawX2, rawY2, x, y, w, h);
if (clipped) {
  const roundedSegments = clipSegmentToRoundedRect(
    clipped.x1, clipped.y1, clipped.x2, clipped.y2, x, y, w, h, fillCornerRadii
  );
  // keep only non-trivial segments
}
```

### 5) Render-Time Clip To Prevent Stroke Bleed
Even after geometric clipping, stroke caps/weight can extend slightly. A canvas clip path enforces rounded boundaries during drawing.

```js
function drawFillSegmentPortion(section, segment, fromDistance, toDistance) {
  const clipRadii = section.fillCornerRadii || getInsetCornerRadii(section.cornerRadii, inset, w, h);
  withRoundedRectClip(x, y, w, h, clipRadii, () => {
    drawSegmentPortion(segment, fromDistance, toDistance);
  });
}
```

### 6) Two-Phase Progressive Animation
Outlines are drawn first; fills begin only after outline segments are exhausted.

```js
if (drawPhase === "outlines") {
  const outlineConsumed = drawNextOutlineDistance(remainingDistance, isHumanSpeed);
  if (outlineConsumed === 0) drawPhase = "fills";
}
```

### 7) Speed Integration With Main App
Distance budget per frame is delegated to app speed profiles so `1x` stays hand-drawn and higher speeds complete faster.

```js
const isHumanSpeed = getMainSpeedMode() === 1;
let remainingDistance = getDrawDistancePerFrame();
if (isHumanSpeed) {
  remainingDistance = min(remainingDistance, PIXEL_STEP_DISTANCE * HUMAN_PIXEL_STEPS_PER_FRAME);
}
```
